- `client_manager` — clients module full access

Enforce permissions on every API route and hide UI nav items by role.

## DATABASE SCHEMA (Prisma models — implement fully)

**User**
- id, email, passwordHash, fullName, avatarUrl, role (enum), teamId, jobTitle, status (active/inactive), createdAt

**Team**
- id, name (enum: DEVELOPER, MARKETING, MANAGEMENT, FINANCE, CLIENT_MGMT, AI), color hex

**Project**
- id, name, description, teamId, priority (HIGH/MEDIUM/LOW), status (TODO/IN_PROGRESS/REVIEW/DONE), progress (0-100), deadline, budget (decimal), clientId optional, createdById

**Task**
- id, title, projectId optional, assigneeId, teamId, priority, dueDate, status (PENDING/DONE), createdById

**EmployeeProfile** (extends User or 1:1)
- baseSalary, bonus, payrollStatus (PENDING/APPROVED/PROCESSED), bankDetails optional (encrypted or omitted in v1)

**Client**
- id, contactName, companyName, email, phone, assignedTeamId, status (ACTIVE/NEGOTIATION/CHURNED), contractValue, nextMeetingAt

**ClientProject** (link client ↔ project)

**Approval**
- id, type (SALARY_HIKE, PROJECT_BUDGET, EXPENSE_CLAIM, NEW_HIRE, SOFTWARE_LICENSE, OTHER), title, description, amount optional, requestedById, teamId, status (PENDING/APPROVED/REJECTED), reviewedById, reviewedAt

**Expense** (for finance)
- id, description, amount, teamId, submittedById, status, receiptUrl optional

**PayrollRun**
- id, month, year, totalAmount, status, approvedById

**Payslip**
- id, userId, payrollRunId, baseSalary, bonus, deductions, netPay, pdfUrl optional

**Report**
- id, type (FINANCIAL, PROJECT_STATUS, TEAM_PERFORMANCE, CLIENT_REVENUE, PAYROLL, AI_USAGE), title, period, generatedAt, fileUrl optional

**Channel**
- id, name, teamId optional, type (TEAM/ANNOUNCEMENT/DM)

**Message**
- id, channelId, senderId, body, createdAt

**ActivityLog**
- id, actorId, action, entityType, entityId, metadata JSON, createdAt (powers dashboard "Recent Activity")

**Notification**
- id, userId, title, body, read, link

Add indexes on foreign keys and `(status, teamId)` where queried often.

## API ENDPOINTS (implement all with auth + validation)

### Auth
- POST `/api/v1/auth/register` (super_admin only after first user)
- POST `/api/v1/auth/login`
- POST `/api/v1/auth/logout`
- GET `/api/v1/auth/me`

### Dashboard
- GET `/api/v1/dashboard` → activeProjectsCount, employeeCount, monthlyRevenue, pendingApprovals, recentActivity[], upcomingDeadlines[], teamLoad[]

### Projects
- CRUD `/api/v1/projects`
- PATCH `/api/v1/projects/:id/status` (for Kanban drag or status dropdown)
- PATCH `/api/v1/projects/:id/progress`

### Tasks
- CRUD `/api/v1/tasks`
- PATCH `/api/v1/tasks/:id/toggle`

### Employees / Teams
- GET `/api/v1/teams`
- GET `/api/v1/teams/:id/members`
- CRUD `/api/v1/employees` (finance + management only for salary fields)

### Finance
- GET `/api/v1/finance/summary` (payroll, revenue, expenses, net profit)
- GET `/api/v1/finance/salaries`
- POST `/api/v1/finance/payroll-runs`
- POST `/api/v1/finance/payslips/:userId/send`
- CRUD `/api/v1/finance/expenses`

### Clients
- CRUD `/api/v1/clients`

### Approvals
- GET `/api/v1/approvals?status=PENDING`
- POST `/api/v1/approvals`
- POST `/api/v1/approvals/:id/approve`
- POST `/api/v1/approvals/:id/reject`

### Reports
- GET `/api/v1/reports`
- POST `/api/v1/reports/generate` (type + period) → create PDF or CSV stub and return download URL

### Comms
- GET `/api/v1/channels`
- GET `/api/v1/channels/:id/messages`
- POST `/api/v1/channels/:id/messages`
- WebSocket: `message:new`, `notification:new`

### Activity
- GET `/api/v1/activity?limit=20`

## BUSINESS RULES

1. Creating a project with budget > ₹5,00,000 auto-creates a pending **PROJECT_BUDGET** approval for Management.
2. Salary change > 10% requires Finance + Management approval chain.
3. Task completion updates project progress (% of done tasks / total tasks on that project).
4. Dashboard stats are computed from DB, not hardcoded.
5. Employees see only their team's channel + company announcements; Management sees all team channels.
6. Audit: every approve/reject and salary change writes to ActivityLog.
